# RenderMesh Local Lab

This directory contains a complete local lab for exercising RenderMesh with:

- MinIO as an S3-compatible bucket.
- A seeded frontend bucket.
- A tiny local edge API.
- A RenderMesh manifest with one origin and wildcard host mapping.

## Start Dependencies

From the repository root:

```bash
docker compose up -d jaeger minio edge-api
docker compose run --rm minio-init
```

MinIO console:

- URL: `http://localhost:9001`
- user: `rendermesh`
- password: `rendermesh-secret`

## Run RenderMesh

Run RenderMesh on the host so it can call the edge API at `127.0.0.1:4010`:

```bash
export RENDERMESH_MANIFEST=./examples/local/rendermesh.yaml
export LOCAL_APP_STORAGE_ENDPOINT=http://127.0.0.1:9000
export LOCAL_APP_STORAGE_REGION=us-east-1
export LOCAL_APP_ACCESS_KEY_ID=rendermesh
export LOCAL_APP_SECRET_ACCESS_KEY=rendermesh-secret
export LOCAL_APP_FORCE_PATH_STYLE=true
export APP_HOST=127.0.0.1
export APP_PORT=3000
export OTEL_ENABLED=false

cargo run
```

Use `Host: test.com` for RenderMesh requests:

```bash
curl -i -H 'Host: test.com' http://127.0.0.1:3000/
```

For browser testing, add this entry to `/etc/hosts`:

```text
127.0.0.1 test.com
```

Then open:

```text
http://test.com:3000/
```

## Flows To Try

### `GET /`

Main RenderMesh flow. The request maps `test.com` to the `local_app` origin, calls the local edge API, receives `params`, and renders `index.html` from the in-memory Handlebars template registry.

```bash
curl -i -H 'Host: test.com' http://127.0.0.1:3000/
```

Expected result: `200 OK`, HTML rendered with edge-provided content, and the `x-rendermesh-edge: params` response header.

### `GET /home`

Explicit rewrite flow. The origin edge config rewrites `/home` to `/index.html`, then the edge API returns `params` and RenderMesh renders the same HTML template used by `/`.

```bash
curl -i -H 'Host: test.com' http://127.0.0.1:3000/home
```

Expected result: `200 OK`, rendered `index.html`.

### `GET /static.html`

Static pass-through flow. The edge API returns only headers and no `params`, so RenderMesh serves `static.html` directly from the local mirror without running Handlebars.

```bash
curl -i -H 'Host: test.com' http://127.0.0.1:3000/static.html
```

Expected result: `200 OK`, `x-rendermesh-edge: static-pass-through`, and the literal `{{title}}` token remains visible in the body.

### `GET /direct`

Direct edge response flow. The edge API returns a terminal `{ "body": "..." }` payload, so RenderMesh does not read a file from the bucket for this request.

```bash
curl -i -H 'Host: test.com' http://127.0.0.1:3000/direct
```

Expected result: `202 Accepted` with the body generated by the edge API.

### `GET /file`

Edge-selected file flow. The edge API returns `{ "file_path": "/static.html" }`, so RenderMesh serves that file from the local mirror.

```bash
curl -i -H 'Host: test.com' http://127.0.0.1:3000/file
```

Expected result: `200 OK`, serving the `static.html` contents selected by the edge API.

### `GET /docs`

Auto-index flow. The origin config enables `auto_rewrite_index`, so `/docs` automatically resolves to `/docs/index.html`.

```bash
curl -i -H 'Host: test.com' http://127.0.0.1:3000/docs
```

Expected result: `200 OK`, serving `docs/index.html`.

### `GET /legacy`

Redirect flow. The origin edge config has a redirect rule from `/legacy` to `/static.html`.

```bash
curl -i -H 'Host: test.com' http://127.0.0.1:3000/legacy
```

Expected result: `302 Found` with `Location: /static.html`.

### `GET /anything-else`

Missing-file flow. Unknown paths use the configured missing behavior:

```yaml
missing:
  action: not_found
  page: /index.html
```

RenderMesh returns the configured page body with a `404 Not Found` status.

```bash
curl -i -H 'Host: test.com' http://127.0.0.1:3000/anything-else
```

Expected result: `404 Not Found` using `/index.html` as the response body.

## Update The Bucket

Edit:

```text
examples/local/bucket/index.html
```

Then reseed MinIO:

```bash
docker compose run --rm minio-init
```

RenderMesh syncs the local mirror every 5 seconds in this lab. Wait a few seconds and request `/` again.

During each successful sync, RenderMesh recompiles the HTML templates for the origin into memory. Requests with edge `params` render from that in-memory registry; `static.html` is still served directly because the edge API does not send params for that route.
